Graffiti Street (2023)

Solo Project - Programming, Art, 3D Modelling

Graffiti Street is the project I worked on for my final dissertation at the University of Sussex. In this project, I aimed to create a game which utilises modern technologies to provide the player with an environment in which they could interact with the game world by drawing onto it. It is for this reason that the theme of graffiti was chosen.

In this project, I handled all the programming, design, modelling and art myself. The project required me to learn how to create my own shaders in Unity through the use of HLSL coding, in which I created the cel shader and outline shader you can see in the accompanying video. As previously mentioned, this project was made with the Unity engine, meaning the majority of the code outside of shaders was written in C#. I also used my existing skills in Blender to create the models which you can see in the game. Graffiti Street is in active development and is being updated outside of the scope of university, now that I have graduated.

You can find my final report for the project here

Caff3.ine (2022)

Group Project - Programming, Environment Design

Caff3.ine was a group project I worked on in my second year at the University of Sussex. We were tasked with an 8 month development cycle, in which time, we were expected to produce a full, working game from conceptualisation to implementation. This project also required us to make a design document for the game.

As this project was done in a group, it taught me key communication and team work skills, as well as giving me an idea of how to work in a professional environment through practices such as code reviews and version control via GitHub.

Throughout this project, I worked on greyboxing the level design, system logic in C#, programming UI and implementing JSON compatibility with the UI.

Snip and Shear (2023)

Group Project - Level Design, Animation, 3D Modelling

Snip and Shear was the submission my group and I made for Global Game Jam 2023, where the theme was "Roots". The team on this project was the same as Caff3.ine.

For this project, I wanted to take a break from programming and develop my skills in other areas, namely 3D modelling and animation, as I had just taught myself Blender and was eager to expand my portfolio. I also lead the level design for this game, designing and modelling the game map entirely by myself. When designing the level, I kept level design principles in mind and took inspiration from shooter games such as Halo and Team Fortress 2 when building the level, which resulted in a level with a large focus on chokepoints and high-risk, high-reward collectables.

Dual Runner (2022)

Group Project - Level Design, Game Design, Animation

Dual Runner was a group submission for Global Game Jam 2022, where the theme was "Duality".

Dual Runner's levels were designed to somewhat emulate that of the classic Sonic the Hedgehog games. In its level design, you can see design which splits the level into multiple paths, with the top path being harder to stay on, but significantly faster, rewarding the player with the sensation of going fast. Additionally, when designing the game's mechanics, I decided to use the level design to attempt to teach the mechanics to the player without a word of dialogue, introducing the player to a mechanic in a safe environment first, then removing the safety net and building on the mechanic to create interesting and challenging level design.

You can download the game here.

Property Tycoon (2022)

Group Project - Programming, UI, Design

Property Tycoon was a second year group project in which we were tasked as a group to create a digital version of the popular game Monopoly.

Recreating Monopoly meant working with an array-based solution in which each space on the game board was its own array object and derived from a parent class which contained information such as the name, property value, position on the board and any purchased properties. This project required us to get this data and load it in from an external file, which would allow for the board to be customised by the user with custom property names and prices, which is why you can see the board loading up placeholder names before the game starts.

Other areas which had to be created included creating a dynamic UI which changes the position of the user's card on the screen depending on the number of players, AI controlled players and the ability to load and save the game through the use of JSON files.

Ground Control (2020)

Solo Project - Level Design, Game Design, Programming

Ground Control was my submission for the GTMK 2020 game jam, where the theme was "Out of Control".

This game was my first fully realised project and my first proper endeavour into Unity and game development. In the game, the player must control a satellite which needs to be positioned over a planet in order to receive data. However, the satellite is malfunctioning and will have a random effect applied to it as it attempts to line itself up. These range from disabled thrusters to corrupted VRAM.
